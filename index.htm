Rules of simple design

  1. Runs all the tests
  2. Contains no duplications
  3. Expresses the intent of the programmers
  4. Has no superfluous parts

Session #2

Ping-Pong TDD

Ping-Pong Programming

  1. Alice writes a new test and sees that it fails
  2. Bob implements the code needed to pass the test
  3. Bob writes the next test and sees that it fails
  4. Alice implements the code needed to pass the test

Session #3

Extreme Normal Form

Extreme Normal Form

  1. The code should work
  2. No bad names
  3. No duplication of code, concepts, ideas
  4. No redundant code - as simple as possible

Session #4

Mute Evil Programmer

Mute Evil Programmer

  • Ping-pong - one person writes the tests, the other person writes the implementation code
  • Mute - nobody can talk
  • Find the loophole - the implementation person purposely writes the wrong algorithm that still makes the tests turn green.
